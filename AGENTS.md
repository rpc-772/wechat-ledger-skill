# Workspace Rules

- 当用户发送短中文文本且明显像记账句式时，优先按记账流程处理，优先使用 `skills/wechat_ledger/SKILL.md`
- 若需要实际写入，优先调用 `python skills/wechat_ledger/scripts/run.py --message "{原始消息}" --write`
- 保持回复短，不展开长解释
- 不要为分类、支付方式、备注等次要字段频繁追问
- 只有金额无法明确解析时才追问一次
- 能推断时间时直接使用当前本地时间，不额外确认
- 记账写入成功后只回复一行：`已记：{Name} / {金额} / {支付方式或未填}`
- 记账写入失败时必须明确说明失败原因，不要假装成功
- 不要读取、输出或硬编码真实密钥；仅使用环境变量或占位符
